Results

GSCF workflow-generation benchmark

The first GROMACS Scientific Code-generation and workflow-generation Framework (GSCF) prototype dataset contained 10 workflow-generation tasks covering common pure-water molecular dynamics operations in GROMACS. These tasks included density calculation, radial distribution function analysis, mean-square displacement and diffusion analysis, hydrogen-bond analysis, z-direction density profiling, coordination-number calculation, equilibration validation, trajectory preprocessing, and temperature time-series extraction.

Across the 10 GSCF tasks, the manual LLM baseline achieved an average evaluator score of 8.700 out of 10, with a minimum score of 8.0 and a maximum score of 9.0. This indicates that the baseline model was generally able to generate syntactically usable and scientifically reasonable GROMACS workflows. However, the results also show that several workflows still required minor correction, especially in analysis-specific validation notes, trajectory-length warnings, atom-selection details, and reproducible command handling.

GEQS evaluator benchmark

The GROMACS Evaluation Quality Set (GEQS) prototype dataset contained 10 evaluator records representing common GROMACS workflow errors. These included missing topology preparation, weak barostat selection, invalid GROMACS command options, missing short-trajectory warnings, wrong RDF atom selection, wrong density-profile command usage, unsuitable trajectory preprocessing for MSD analysis, incomplete equilibration validation, incorrect solvent count in topology files, and non-reproducible interactive selection handling.

The evaluator achieved exact agreement with the expert score for all 10 GEQS records. The mean absolute error was 0.000, the mean squared error was 0.000, and the root mean squared error was 0.000. This result suggests that the evaluator prompt and rubric were able to consistently identify key GROMACS-specific workflow errors in the current prototype benchmark.

Evaluator-guided correction-loop performance

A correction-loop test was performed on five selected GSCF tasks. The average correction improvement was 0.600 score points, and three out of five workflows improved after evaluator-guided revision. The strongest improvement was observed for workflows with clear methodological weaknesses, such as MSD diffusion analysis, density-profile analysis, and trajectory preprocessing. Already strong workflows remained stable after correction.

These results show that evaluator-guided correction can reduce common GROMACS workflow-generation errors without degrading already acceptable outputs. The correction loop was especially useful for improving reproducibility, command specificity, validation notes, and analysis-specific caveats.

Overall prototype outcome

The first prototype demonstrates that an LLM-based GROMACS workflow agent can generate high-quality workflows for common molecular dynamics analysis tasks, while an evaluator module can identify GROMACS-specific errors with strong consistency. The current results support the feasibility of building a larger GROMACS workflow-generation and evaluation benchmark. However, this stage remains a prototype. A publication-ready version should expand the dataset beyond 10 GSCF and 10 GEQS tasks, include more molecular systems, compare multiple LLM baselines, and validate selected workflows by actual GROMACS execution.

# 3. Topology preparation steps

Even though temperature extraction uses `.edr` files, topology preparation should not be skipped in a reproducible workflow.

For a pure water system, the topology should define the water model and the final number of water molecules.

Example `topol.top` structure:

```text
#include "oplsaa.ff/forcefield.itp"
#include "oplsaa.ff/tip3p.itp"

[ system ]
Pure water system

[ molecules ]
SOL   1000
```

Before analysis, check that:

```bash
ls topol.top
```

Also inspect the topology:

```bash
grep -A 5 "\[ molecules \]" topol.top
```

The number of `SOL` molecules in `topol.top` should match the final solvated system used to generate the `.edr` files.

Important note:

```text
topol.top is not directly read by gmx energy, but it is required for reproducibility because it documents the molecular system used to generate nvt.edr, npt.edr, and md.edr.
```

# 10. Important interpretation note

Temperature stability alone does **not** prove full equilibration.

To fully validate equilibration, also check:

```text
Density
Pressure
Total energy
Potential energy
Box volume
RMSD or structural stability, if relevant
```

For pure water at 300 K and 1 bar, the temperature should remain near 300 K, but pressure and density must also be checked before claiming complete equilibration.

If the trajectory is very short, the result should be reported only as:

```text
workflow validation
```

not as final proof of physical convergence.

# 11. Common errors and fixes

## Error 1: Wrong energy file used

Example mistake:

```bash
echo Temperature | gmx energy -f em.edr -o temperature.xvg
```

Problem:

```text
Energy minimization does not represent a temperature-equilibrated MD trajectory.
```

Fix:

Use NVT, NPT, or production MD energy files:

```bash
echo Temperature | gmx energy -f nvt.edr -o temperature_nvt.xvg
echo Temperature | gmx energy -f npt.edr -o temperature_npt.xvg
echo Temperature | gmx energy -f md.edr -o temperature_md.xvg
```

---

## Error 2: Interactive selection not handled

If you run:

```bash
gmx energy -f md.edr -o temperature_md.xvg
```

GROMACS asks you to select an energy term manually.

Fix:

Use non-interactive echo-based selection:

```bash
echo Temperature | gmx energy -f md.edr -o temperature_md.xvg
```

---

## Error 3: Temperature option name not recognized

Some systems may show slightly different energy-term names.

Fix:

First list available energy terms:

```bash
gmx energy -f md.edr
```

Then select the correct temperature term manually, or use the displayed number.

Example:

```bash
echo 16 | gmx energy -f md.edr -o temperature_md.xvg
```

Only use the number after confirming that it corresponds to **Temperature**.

---

## Error 4: Missing `.edr` file

Example error:

```text
File input/output error:
md.edr not found
```

Fix:

Check files:

```bash
ls *.edr
```

If `md.edr` is missing, rerun production MD:

```bash
gmx mdrun -deffnm md
```

---

## Error 5: Assuming temperature stability proves equilibration

Incorrect statement:

```text
The system is fully equilibrated because temperature is stable.
```

Correct statement:

```text
The temperature is stable around 300 K, but full equilibration also requires checking density, pressure, total energy, and box volume.
```

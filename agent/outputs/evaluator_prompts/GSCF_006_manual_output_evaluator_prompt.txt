# Complete GROMACS Workflow: O–H RDF of Pure Water at 300 K

## 1. Objective

Calculate the radial distribution function between water oxygen atoms and water hydrogen atoms in pure water at **300 K and 1 bar** using GROMACS.

Target RDF:

```bash
gmx rdf -f md.xtc -s md.tpr -o rdf_OW_HW.xvg -ref "name OW" -sel "name HW"
```

Expected main output:

```bash
rdf_OW_HW.xvg
```

---

## 2. Required Input Files

For the final RDF analysis, the minimum required files are:

```text
md.tpr
md.xtc
topol.top
```

For the full reproducible workflow, prepare or generate:

```text
water_box.gro
topol.top
ions.mdp or minim.mdp
nvt.mdp
npt.mdp
md.mdp
```

If starting from scratch, the water box can be generated using GROMACS built-in water coordinates.

---

## 3. Topology Preparation

Use a standard GROMACS-compatible water model such as **SPC/E**, **SPC**, or **TIP3P**.

Example using SPC/E water:

Create `topol.top`:

```text
#include "oplsaa.ff/forcefield.itp"
#include "oplsaa.ff/spce.itp"

[ system ]
Pure water box for O-H RDF analysis at 300 K

[ molecules ]
SOL  0
```

After solvation, update the number of water molecules in `[ molecules ]`.

For example, if GROMACS reports that 216 molecules were added:

```text
[ molecules ]
SOL  216
```

Important: check the actual water atom names in your `.gro` file. Common names are:

```text
OW   water oxygen
HW1  first water hydrogen
HW2  second water hydrogen
```

Some datasets or prepared water files may use:

```text
OW
HW
```

The validated dataset command uses:

```bash
-ref "name OW" -sel "name HW"
```

If your hydrogens are named `HW1` and `HW2`, use:

```bash
-sel "name HW1 HW2"
```

or rename atoms consistently before analysis.

---

## 4. Build Pure Water Simulation Box

Create a cubic water box:

```bash
gmx solvate -cs spc216.gro -o water_box.gro -box 3.0 3.0 3.0 -p topol.top
```

This creates a pure water box of approximately 3 nm × 3 nm × 3 nm.

Check the number of water molecules:

```bash
tail -n 20 topol.top
```

Expected topology ending should look like:

```text
[ molecules ]
SOL   number_of_water_molecules
```

## 7.3 Optional: Intermolecular O–H RDF

The simple command:

```bash
gmx rdf -f md.xtc -s md.tpr -o rdf_OW_HW.xvg -ref "name OW" -sel "name HW"
```

may include both:

```text
intramolecular O-H correlations
intermolecular O-H correlations
```

For water, the intramolecular O–H bond produces a sharp peak near the covalent O–H distance, around **0.1 nm**. This peak is not the hydrogen-bond structure between different water molecules.

If you want mainly intermolecular O–H RDF, use topology exclusions:

```bash
gmx rdf -f md.xtc -s md.tpr -o rdf_OW_HW_intermolecular.xvg -ref "name OW" -sel "name HW" -excl
```

If your hydrogens are named `HW1` and `HW2`:

```bash
gmx rdf -f md.xtc -s md.tpr -o rdf_OW_HW_intermolecular.xvg -ref "name OW" -sel "name HW1 HW2" -excl
```

Report clearly which RDF was used:

```text
rdf_OW_HW.xvg includes all O-H correlations unless exclusions are applied.
rdf_OW_HW_intermolecular.xvg excludes bonded intramolecular O-H pairs and is more suitable for liquid-water hydrogen-bond structural interpretation.
```

# 10. Interpretation of O–H RDF

For pure water:

The O–H RDF describes how hydrogen atoms are distributed around water oxygen atoms.

Important interpretation:

```text
A sharp peak near 0.1 nm usually represents intramolecular covalent O-H bonds if bonded pairs are not excluded.
A broader peak around hydrogen-bonding distance represents intermolecular water structure.
```

Therefore, for structural hydrogen-bond analysis, it is better to report the intermolecular RDF using:

```bash
-excl
```

or another method that excludes hydrogens belonging to the same water molecule.

# 11. Common Possible Errors and Fixes

## Error 1: Missing `md.tpr`

Problem:

```text
Fatal error: File md.tpr does not exist
```

Fix:

Run production preprocessing first:

```bash
gmx grompp -f md.mdp -c npt.gro -t npt.cpt -p topol.top -o md.tpr
```

---

## Error 2: Missing `md.xtc`

Problem:

```text
Fatal error: File md.xtc does not exist
```

Fix:

Run production MD:

```bash
gmx mdrun -deffnm md
```

Also check that `nstxout-compressed` in `md.mdp` is not zero.

---

## Error 3: Wrong atom names

Problem:

```text
Selection 'name HW' does not match any atoms
```

Fix:

Check atom names:

```bash
head -n 30 md.gro
```

If hydrogens are named `HW1` and `HW2`, use:

```bash
gmx rdf -f md.xtc -s md.tpr -o rdf_OW_HW.xvg -ref "name OW" -sel "name HW1 HW2"
```

---

## Error 4: Unclear interactive group selection

Problem: using old-style index groups can cause wrong selections.

Avoid this:

```bash
gmx rdf -f md.xtc -s md.tpr -o rdf.xvg
```

Better:

```bash
gmx rdf -f md.xtc -s md.tpr -o rdf_OW_HW.xvg -ref "name OW" -sel "name HW"
```

This is clearer and reproducible.

---

## Error 5: Intramolecular O–H peak misinterpreted

Problem:

The RDF shows a large sharp peak near 0.1 nm, and it is incorrectly interpreted as intermolecular hydrogen bonding.

Fix:

Explain that the default O–H RDF may include intramolecular O–H covalent bonds. For intermolecular water structure, use:

```bash
gmx rdf -f md.xtc -s md.tpr -o rdf_OW_HW_intermolecular.xvg -ref "name OW" -sel "name HW" -excl
```

---

## Error 6: Short trajectory treated as final result

Problem:

A 100 ps or very short production trajectory is used and the RDF is reported as publication-quality.

Fix:

Write clearly:

```text
Because the production trajectory is short, the RDF result should be treated only as workflow validation. A longer trajectory, such as several nanoseconds or more, is required for reliable publication-quality RDF statistics.
```

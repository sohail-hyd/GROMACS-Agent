2. Topology preparation
2.1 Create topology before solvation

Create topol.top first:

nano topol.top

Add:

#include "amber99sb-ildn.ff/forcefield.itp"
#include "amber99sb-ildn.ff/spce.itp"

[ system ]
Pure SPC/E water box for self-diffusion at 300 K

[ molecules ]

This is safer because gmx solvate -p topol.top can only update the topology if the topology file already exists.

2.2 Create pure water box

Create a 3 nm × 3 nm × 3 nm water box:

gmx solvate -cs spc216.gro -box 3.0 3.0 3.0 -o water_box.gro -p topol.top

After running this command, GROMACS should update topol.top automatically by adding a line like:

SOL  895

The exact number may be different. Use the number printed by gmx solvate as the primary source.

Check the end of topol.top:

tail topol.top

It should look like:

[ molecules ]
SOL  895

Do not rely only on:

grep SOL water_box.gro | wc -l

because this counts atom lines, not water molecules directly. For SPC/E water, each water molecule has 3 atoms, so the molecule number would be total SOL atom lines divided by 3, but the safer method is to use the count printed by gmx solvate.

5. MSD post-processing workflow
5.1 Remove periodic boundary jumps

For MSD analysis, the trajectory should be treated so molecules move continuously across periodic boundaries.

echo "System" | gmx trjconv -s md.tpr -f md.xtc -o md_nojump.xtc -pbc nojump

Expected file:

md_nojump.xtc
5.2 Create index group for water oxygen atoms

For rigid water models such as SPC/E, the oxygen atom is commonly used as a molecular tracer because it represents the position of each water molecule consistently. This is acceptable for water self-diffusion analysis.

Create index file:

gmx make_ndx -f md.gro -o index.ndx

Inside the interactive menu, type:

a OW
name 3 Water_Oxygen
q

If group number 3 is already used differently, check the group number printed by GROMACS and rename the correct new group.

If the oxygen atom name is not OW, check the coordinate file:

head water_box.gro

Common oxygen names are:

OW
OH2
O
5.3 Calculate MSD and diffusion coefficient

Use:

echo "Water_Oxygen" | gmx msd -s md.tpr -f md_nojump.xtc -n index.ndx -o msd_water.xvg -beginfit 1000 -endfit 9000

Do not use invalid options such as:

-type no

The diffusion coefficient will be printed in the terminal and written in the comments of:

msd_water.xvg

The fitting range should be inside the approximately linear region of the MSD curve. For a 10 ns trajectory, 1000–9000 ps is a reasonable first fitting window, but you should test other windows too.

Example:

echo "Water_Oxygen" | gmx msd -s md.tpr -f md_nojump.xtc -n index.ndx -o msd_water_fit2.xvg -beginfit 2000 -endfit 9000
5.4 Optional molecule-based MSD note

For rigid water, oxygen MSD is normally acceptable because each oxygen tracks one water molecule. A true center-of-mass water MSD may also be used if your GROMACS version and index setup support molecule-based COM analysis, but do not add unsupported options.

For this workflow, the recommended reproducible command is the oxygen-tracer MSD command above.

5.5 View MSD curve
xmgrace msd_water.xvg

Or copy the data for plotting in Origin, Excel, Python, or other software:

cp msd_water.xvg water_msd_300K.xvg

The MSD curve should be nonlinear at very short times, then become approximately linear during the diffusive regime.

7. Common errors and fixes
Error 1: topol.top missing

Message:

Fatal error:
Topology file topol.top not found

Fix:

Create topol.top before running gmx solvate -p topol.top.

Error 2: coordinate-topology mismatch

Message:

Number of coordinates in coordinate file does not match topology

Fix:

Check the final [ molecules ] section:

tail topol.top

The number of SOL molecules must match the number inserted by gmx solvate.

Error 3: inconsistent water model

Problem:

Using spc216.gro but including tip3p.itp may cause model inconsistency.

Fix:

Use consistent SPC/E setup:

coordinate source: spc216.gro
topology include: spce.itp
molecule name: SOL
oxygen atom: OW
Error 4: energy command stops or waits for input

Problem:

Some GROMACS versions need a terminating 0.

Less reliable:

echo "Temperature" | gmx energy -f nvt.edr -o temperature_nvt.xvg

Better:

echo -e "Temperature\n0" | gmx energy -f nvt.edr -o temperature_nvt.xvg
Error 5: wrong oxygen atom name

Message:

Atom type OW not found

Fix:

Check atom names:

head water_box.gro

Then use the correct oxygen name in gmx make_ndx, for example:

a OW

or:

a OH2

or:

a O
Error 6: MSD curve is not linear

Possible causes:

trajectory too short
system not equilibrated
wrong fitting interval
PBC jumps not corrected
temperature or density unstable

Fix:

Use a longer trajectory, for example 10 ns or more, and test different fitting windows:

echo "Water_Oxygen" | gmx msd -s md.tpr -f md_nojump.xtc -n index.ndx -o msd_water_fit2.xvg -beginfit 2000 -endfit 9000
Error 7: unrealistic diffusion coefficient

Possible causes:

short trajectory
poor equilibration
wrong water model
wrong PBC treatment
wrong fitting range
temperature not near 300 K
density not equilibrated

Fix:

Check:

echo -e "Temperature\n0" | gmx energy -f md.edr -o temperature_md.xvg
echo -e "Density\n0" | gmx energy -f md.edr -o density_md.xvg

Then repeat MSD with different fitting windows.

Error 8: invalid MSD option

Wrong:

gmx msd -type no

Correct:

echo "Water_Oxygen" | gmx msd -s md.tpr -f md_nojump.xtc -n index.ndx -o msd_water.xvg -beginfit 1000 -endfit 9000

3. Topology preparation steps

Even though this task focuses on trajectory preprocessing, topology preparation should not be skipped in a complete workflow.

For a pure water system, the topology should already have been prepared before the MD run. A typical topology preparation sequence is:

gmx pdb2gmx -f water_initial.pdb -o water_processed.gro -p topol.top

Select a suitable water model, for example SPC/E, TIP3P, or TIP4P depending on the force field.

Then define the box:

gmx editconf -f water_processed.gro -o water_box.gro -c -d 1.0 -bt cubic

Solvate the box:

gmx solvate -cp water_box.gro -cs spc216.gro -o water_solv.gro -p topol.top

Prepare and run minimization, NVT, NPT, and production MD:

gmx grompp -f minim.mdp -c water_solv.gro -p topol.top -o em.tpr
gmx mdrun -deffnm em

gmx grompp -f nvt.mdp -c em.gro -p topol.top -o nvt.tpr
gmx mdrun -deffnm nvt

gmx grompp -f npt.mdp -c nvt.gro -p topol.top -o npt.tpr
gmx mdrun -deffnm npt

4. GROMACS command sequence for trajectory preprocessing
Step 1: Remove large periodic jumps
echo System | gmx trjconv -s md.tpr -f md.xtc -o md_nojump.xtc -pbc nojump

Purpose:

-pbc nojump removes large discontinuous jumps caused by molecules crossing the periodic box boundary.

Output:

md_nojump.xtc

Important note:

This does not overwrite md.xtc. The original trajectory is preserved.
Step 2: Make water molecules whole
echo System | gmx trjconv -s md.tpr -f md_nojump.xtc -o md_whole.xtc -pbc mol

Purpose:

-pbc mol reconstructs molecules so that atoms belonging to the same molecule are kept together.

Output:

md_whole.xtc

This is important because water molecules can appear split across periodic boundaries in the raw trajectory.

Step 3: Center the system and keep molecules whole

For a homogeneous pure water system, centering the whole System group is acceptable:

echo System System | gmx trjconv -s md.tpr -f md_whole.xtc -o md_center.xtc -center -pbc mol

A safer Bash version is:

printf "System\nSystem\n" | gmx trjconv -s md.tpr -f md_whole.xtc -o md_center.xtc -center -pbc mol

Purpose:

-center centers the selected group in the simulation box.
-pbc mol keeps molecules whole after centering.

Group selection explanation:

First selection: group to center.
Second selection: group to write to the output trajectory.

For pure water:

Center group: System
Output group: System

For solute-water systems, do not blindly center System. Usually, select the solute for centering and select System for output:

printf "Protein\nSystem\n" | gmx trjconv -s md.tpr -f md_whole.xtc -o md_center.xtc -center -pbc mol

7. Common possible errors and fixes
Error 1: Original trajectory overwritten

Wrong:

gmx trjconv -s md.tpr -f md.xtc -o md.xtc -pbc nojump

Fix:

gmx trjconv -s md.tpr -f md.xtc -o md_nojump.xtc -pbc nojump

Always write a new trajectory file.

Error 2: Wrong group selection during centering

Problem:

The wrong group is selected for centering or output.

Fix:

For pure water:

printf "System\nSystem\n" | gmx trjconv -s md.tpr -f md_whole.xtc -o md_center.xtc -center -pbc mol

For solute-water systems:

printf "Solute\nSystem\n" | gmx trjconv -s md.tpr -f md_whole.xtc -o md_center.xtc -center -pbc mol
Error 3: Using the same centering logic for every system

Problem:

Centering System is acceptable for homogeneous pure water, but not always correct for systems containing protein, polymer, nanoparticle, membrane, or solute.

Fix:

For pure water: center System.
For solute-water systems: center solute, output System.
Error 4: Broken molecules still appear after correction

Possible cause:

PBC correction order was wrong or the trajectory was already highly fragmented.

Fix:

Use this order:

-pbc nojump
-pbc mol
-center -pbc mol

Recommended sequence:

echo System | gmx trjconv -s md.tpr -f md.xtc -o md_nojump.xtc -pbc nojump
echo System | gmx trjconv -s md.tpr -f md_nojump.xtc -o md_whole.xtc -pbc mol
printf "System\nSystem\n" | gmx trjconv -s md.tpr -f md_whole.xtc -o md_center.xtc -center -pbc mol
Error 5: Using cleaned trajectory with mismatched .tpr

Problem:

The cleaned trajectory is used with a different or incompatible md.tpr file.

Fix:

Use the same .tpr file that corresponds to the original production trajectory:

gmx rdf -s md.tpr -f md_center.xtc -o rdf.xvg
Error 6: Not visually checking the corrected trajectory

Problem:

Analysis is performed immediately without checking whether the corrected trajectory is physically reasonable.

Fix:

vmd md.gro md_center.xtc

or extract a frame:

gmx trjconv -s md.tpr -f md_center.xtc -o check_frame.gro -dump 0

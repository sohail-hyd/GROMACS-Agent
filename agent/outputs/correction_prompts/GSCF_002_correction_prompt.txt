2. Topology preparation

Create a file named:

nano topol.top

Paste this:

#include "oplsaa.ff/forcefield.itp"

[ system ]
Pure SPC/E water box at 300 K

Do not manually add the number of water molecules yet.
gmx solvate -p topol.top will automatically add the correct number of SOL molecules.

6. Post-processing: O–O RDF calculation

The oxygen atom name for SPC/E water is usually:

OW

Calculate the oxygen–oxygen RDF:

gmx rdf -f md.xtc -s md.tpr -o rdf_OW_OW.xvg -ref 'name OW' -sel 'name OW' -bin 0.002 -rmax 1.2

Expected output:

rdf_OW_OW.xvg

This file contains two main columns:

r / nm      g_OO(r)

Example:

0.270       2.8
0.272       2.9
0.274       3.0

The first major peak of liquid water O–O RDF is usually near:

0.27–0.28 nm
7. Optional: remove early production time

If the beginning of the production trajectory is still relaxing, remove the first 100 ps:

gmx rdf -f md.xtc -s md.tpr -o rdf_OW_OW_after100ps.xvg -ref 'name OW' -sel 'name OW' -bin 0.002 -rmax 1.2 -b 100

Expected output:

rdf_OW_OW_after100ps.xvg
8. Plot RDF result

You can quickly view the RDF using Grace:

xmgrace rdf_OW_OW.xvg

Or convert the .xvg file for plotting in Origin, Excel, or Python:

grep -v "^[@#]" rdf_OW_OW.xvg > rdf_OW_OW.dat

Fix:

Run solvate with -p topol.top:

gmx solvate -cs spc216.gro -o water_box.gro -p topol.top -box 3.0 3.0 3.0

tail topol.top
Error 3: Pressure coupling warning in NVT

Problem:
Pressure coupling is used during NVT.

Fix:
For NVT, use:

pcoupl = no

Pressure coupling should only be used in NPT and production if constant pressure is needed.

Box is too small
Poor equilibration

Fix:

Use a longer production run:

5–20 ns

and calculate RDF after removing early trajectory time:

-b 100

or more, depending on equilibration.

User task:
Generate a complete GROMACS workflow to calculate the density of pure water at 300 K and 1 bar.


Generated baseline workflow:
1. Aim

Calculate the average density of pure water at:

Temperature = 300 K
Pressure    = 1 bar
Ensemble    = NPT
Water model = SPC/E
Box size    = 3.0 × 3.0 × 3.0 nm

This is suitable as a workflow validation run. For publishable density, use a longer production run, for example 5–20 ns after equilibration.

2. Required input files

You need these files in one folder:

topol.top
minim.mdp
nvt.mdp
npt.mdp
md.mdp

You do not need to manually prepare a water .pdb file, because GROMACS can fill a box using spc216.gro.

3. Topology preparation

Create a file named:

nano topol.top

Paste this:

#include "oplsaa.ff/forcefield.itp"
#include "oplsaa.ff/spce.itp"

[ system ]
Pure SPC/E water box

[ molecules ]

Important: at this stage, leave [ molecules ] empty. gmx solvate will automatically add the number of water molecules if -p topol.top is used.

4. Create the water box

Use a 3 nm cubic box:

gmx solvate -cs spc216.gro -box 3.0 3.0 3.0 -o water_box.gro -p topol.top

Expected output:

water_box.gro
topol.top

After this command, check the bottom of topol.top. It should look like:

[ molecules ]
SOL    894

The exact number may be slightly different depending on GROMACS version/settings.

6. GROMACS command sequence
6.1 Energy minimization
gmx grompp -f minim.mdp -c water_box.gro -p topol.top -o em.tpr
gmx mdrun -deffnm em

Expected output:

em.gro
em.edr
em.log
em.tpr

Check potential energy:

echo "Potential" | gmx energy -f em.edr -o potential.xvg
6.2 NVT equilibration
gmx grompp -f nvt.mdp -c em.gro -r em.gro -p topol.top -o nvt.tpr
gmx mdrun -deffnm nvt

Expected output:

nvt.gro
nvt.edr
nvt.log
nvt.xtc

Check temperature:

echo "Temperature" | gmx energy -f nvt.edr -o temperature_nvt.xvg

Expected: temperature should fluctuate around 300 K.

6.3 NPT equilibration
gmx grompp -f npt.mdp -c nvt.gro -t nvt.cpt -p topol.top -o npt.tpr
gmx mdrun -deffnm npt

Expected output:

npt.gro
npt.edr
npt.log
npt.xtc
npt.cpt

Check pressure:

echo "Pressure" | gmx energy -f npt.edr -o pressure_npt.xvg

Check density:

echo "Density" | gmx energy -f npt.edr -o density_npt.xvg

Pressure will fluctuate strongly in a small water box. That is normal. Density should become stable after the box volume equilibrates.

6.4 Production run
gmx grompp -f md.mdp -c npt.gro -t npt.cpt -p topol.top -o md.tpr
gmx mdrun -deffnm md

Expected output:

md.edr
md.log

7. Density analysis

Use gmx energy:

echo "Density" | gmx energy -f md.edr -o density.xvg

The terminal will print something like:

Statistics over 2500001 steps
Energy                      Average   Err.Est.       RMSD  Tot-Drift
Density                     995.xxx   ...

The important value is:

Average Density = around 990–1000 kg/m³

For SPC/E water at 300 K and 1 bar, a reasonable value should be close to experimental water density, but the exact value depends on force field, cutoff settings, box size, equilibration length, and production time.

8. Optional: extract only equilibrated part

For better density averaging, ignore early production time. Example: use only the last 4 ns of a 5 ns run.

echo "Density" | gmx energy -f md.edr -b 1000 -o density_last4ns.xvg

Here:

-b 1000

means start analysis after 1000 ps = 1 ns.

9. Optional MSD analysis

MSD is not required for density, but if you want diffusion validation:

echo "System" | gmx msd -s md.tpr -f md.xtc -o msd_water.xvg

Do not use invalid options such as:

-type no

For pure water, selecting System is acceptable because the whole system is water.

11. Common errors and fixes
Error 1: Invalid order for directive moleculetype

Cause: topology sections are in the wrong order.

Fix: use this order:

#include forcefield
#include water model

[ system ]

[ molecules ]
Error 2: No such moleculetype SOL

Cause: water model .itp file is not included.

Fix: make sure topol.top contains:

#include "oplsaa.ff/spce.itp"
Error 3: number of coordinates does not match topology

Cause: topol.top water molecule number does not match water_box.gro.

Fix: recreate the box using:

gmx solvate -cs spc216.gro -box 3.0 3.0 3.0 -o water_box.gro -p topol.top

Do not manually change the number of water molecules unless you know the exact count.

Error 4: Unknown left-hand mdp entry pcoupl = C-rescale

Cause: your GROMACS version may be old and does not support C-rescale.

Fix: update GROMACS if possible. If not possible, use:

pcoupl = Parrinello-Rahman

For modern GROMACS versions, C-rescale is preferred here.

Error 5: Density is far from 1000 kg/m³

Possible causes:

Production run too short
NPT not equilibrated
Wrong topology/water model
Too small box
Wrong pressure coupling
Analysis included early unstable region

Fix:

echo "Density" | gmx energy -f md.edr -b 1000 -o density_last4ns.xvg

Also increase production time to 10–20 ns for better averaging.

Error 6: Pressure looks very noisy

This is normal for small MD boxes. Pressure fluctuates much more strongly than density. Judge equilibration mainly from:

Stable density
Stable box volume
Stable temperature
No systematic drift
